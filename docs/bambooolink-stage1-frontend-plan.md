# BambooLink 第一阶段前端实施方案（可直接交付 AI 编码工具）

> 角色视角：资深产品经理 + AI 协同开发负责人  
> 分支策略：所有迭代默认在 GitHub `main` branch 直接提交

---

## 1. 第一阶段开发边界

### 1.1 本轮必须完成（In Scope）

**页面范围**
- 官网首页
- 登录页
- 注册页
- 规则中心页

**基础能力范围**
- 全局导航（登录前）
- 全局页脚
- 公共布局容器与栅格
- 基础设计 token（色彩、字体、间距、圆角、边框、阴影）
- 基础通用组件（按钮、卡片、表单项、状态标签）
- 响应式适配（桌面优先，兼容平板与移动端）

**交互范围**
- 顶部导航锚点跳转/页面跳转
- 登录/注册表单前端校验（必填、格式、错误提示）
- 规则中心分类切换与内容定位
- 首页关键按钮跳转（进入登录/注册/规则）

### 1.2 本轮明确不做（Out of Scope）

- 不接入后端接口
- 不生成数据库迁移脚本
- 不实现真实鉴权与会话
- 不实现登录后业务系统（工作台、资源中心、询价、订单、管理后台）
- 不实现 AI 匹配算法，只保留首页能力预告区块
- 不实现支付、消息推送、文件上传

### 1.3 完成定义（DoD）

- 四个页面在统一布局与视觉规范下可正常访问。
- 公共组件复用率高（避免页面内重复样式实现）。
- 文案口径一致，符合产业互联网正式网站语气。
- 全站仅使用正式商用口径措辞。

## 2. 第一阶段页面清单（目标 + 必要性）

### 2.1 首页
- **页面目标**：建立 BambooLink 的业务认知与可信机制认知，并引导用户进入注册/登录/规则中心。
- **为什么第一阶段必须做**：这是正式网站入口，决定平台定位是否被正确理解。

### 2.2 登录页
- **页面目标**：为既有用户提供统一访问入口。
- **为什么第一阶段必须做**：没有登录入口，网站无法形成可持续使用路径。

### 2.3 注册页
- **页面目标**：完成新用户入驻与角色选择（竹农/竹企/管理员）。
- **为什么第一阶段必须做**：平台冷启动依赖角色化注册。

### 2.4 规则中心页
- **页面目标**：公开平台规则（审核、履约、争议、信用），建立制度透明性。
- **为什么第一阶段必须做**：产业协作平台的信任基础来自规则透明。

## 3. 全局布局方案

### 3.1 顶部导航结构（登录前）

**左侧**：Logo + `BambooLink`  
**中部**：平台价值 / 业务流程 / 角色入口 / 规则中心  
**右侧**：`登录`（次按钮） `注册`（主按钮）

**交互建议**
- 首页内使用锚点滚动；跨页使用标准路由跳转。
- 滚动时导航可增加轻微背景与阴影，提高可读性。

### 3.2 页脚结构
- 平台简介（一句话）
- 快速链接（首页、规则中心、登录、注册）
- 联系方式（邮箱/电话占位）
- 法务信息（服务条款、隐私政策占位）
- 版权信息：`© BambooLink. All rights reserved.`

### 3.3 公共容器宽度与栅格
- 最大内容宽度：`1200px`
- 大屏容器：`min(1200px, calc(100% - 64px))`
- 中屏容器：`calc(100% - 48px)`
- 小屏容器：`calc(100% - 32px)`
- 栅格建议：12 栏；卡片间距 20~24px

### 3.4 页面留白建议
- 首屏上下留白：80~96px
- 区块间距：56~72px
- 卡片内边距：20~24px
- 表单区块内边距：24~28px

### 3.5 公共按钮层级
- **Primary**：主路径动作（注册、提交）
- **Secondary**：辅助动作（登录、查看规则）
- **Ghost/Text**：低优先级动作（返回、了解更多）

### 3.6 公共信息区块风格
- 白底卡片 + 1px 浅边框 + 轻阴影
- 标题清晰、描述克制、避免大段密集文本
- 区块结构统一：标题 / 描述 / 内容 / 动作

## 4. 设计系统建议（基础版）

### 4.1 色彩方向（低饱和）

**主色（Green）**
- `#5F7F72`（主品牌绿，低饱和）
- `#6F8F82`（主色悬停）
- `#EAF1EE`（主色浅背景）

**辅助色（Blue-Gray）**
- `#607080`（辅助信息）
- `#E9EEF2`（辅助背景）

**中性色（Gray）**
- `#1F2A24`（正文主文字）
- `#4B5A52`（次级文字）
- `#8A978F`（说明文字）
- `#D7DED9`（边框）
- `#F6F8F7`（页面背景）
- `#FFFFFF`（卡片背景）

**状态色（克制）**
- 成功：`#4F7A61`
- 警告：`#9A7C4A`
- 风险：`#A05A5A`
- 信息：`#5E7485`

### 4.2 页面背景建议
- 全站主背景：`#F6F8F7`
- 关键区块背景：`#FFFFFF`
- 强调区块可用：`#F1F5F3`

### 4.3 字体层级建议
- H1：36/44，半粗
- H2：28/36，半粗
- H3：22/30，中粗
- Body-L：16/26，常规
- Body-M：14/22，常规
- Caption：12/18，常规

### 4.4 圆角、边框、阴影
- 圆角：卡片 12px，输入框/按钮 10px
- 边框：1px `#D7DED9`
- 阴影：`0 6px 20px rgba(31,42,36,0.06)`

### 4.5 卡片风格
- 强调结构稳定，不做大面积渐变
- 统一抬升层级（hover 仅轻微阴影变化）
- 卡片内容遵循“标题 > 说明 > 数据/动作”

### 4.6 状态标签风格
- 胶囊形标签，字号 12
- 使用浅底 + 深字，不使用高饱和纯色底
- 示例：
  - `审核规范`：浅绿色底 + 深绿色字
  - `争议处理`：浅棕底 + 深棕字

## 5. 首页高保真文案框架

### 5.1 Hero 区

`让竹产业交易协作更加清晰、可追溯、可执行。`

**副标题**  
`BambooLink 连接竹农、竹企与平台治理方，围绕资源发布、询价协同、履约跟踪与争议处理，建立标准化的产业协作流程。`

- 主按钮：`立即注册`
- 次按钮：`登录系统`
- 文本按钮：`查看规则中心`

### 5.2 平台价值模块

**模块标题**  
`BambooLink 聚焦产业协作效率与跨主体互信。`

**说明文案**  
`通过结构化信息、流程留痕与规则治理，降低交易沟通成本与履约不确定性。`

1. `资源信息标准化`：字段完整、状态明确、更新可追踪。  
2. `询价与履约协同`：关键条款确认清晰，节点进度可见。  
3. `规则与信用支撑`：争议处理有依据，信用评价可解释。

- **竹农入口标题**：`我是竹农`

  按钮：`以竹企身份注册`

- **管理员入口标题**：`我是管理员`  
  描述：`执行资源审核、争议处置与平台治理。`  
  按钮：`进入管理入口`

### 5.4 可信机制模块文案

**模块标题**  
`可信机制覆盖关键协作环节。`

**机制项**
- `审核机制`：保障资源信息完整与真实性。
- `版本留存`：报价变更过程可追踪。
- `履约凭证`：节点证明材料统一归档。
- `争议处理`：处理过程透明、结论可复核。

### 5.5 AI 匹配预留模块文案

`智能匹配能力（建设中）`

**说明文案**  
`平台将逐步提供资源推荐与需求匹配能力，并展示推荐依据，支持人工确认与业务复核。`
