# 竹产业交易协作平台（Web 版）产品方案（MVP）

## 1. 产品定位

**一句话定位**  
面向竹产业上下游主体的 **供需撮合 + 交易协作 + 可信治理** Web 平台，以“快速匹配与互信达成”为核心目标，帮助竹农与竹企在资源信息透明、询报价协同、履约进度共识、争议处理留痕中高效完成交易。

**产品边界（区别于普通电商）**
- 不是“低价比货”的商品商城，而是以 **资源可得性、稳定供应、协作效率、履约可信** 为主。
- 不是内容社区，不追求信息流消费，而强调 **结构化资源信息 + 流程协同 + 规则治理**。
- 不是只做撮合，需覆盖“发布—审核—询价—确认—履约—争议—评价”闭环。

**Web 化价值**
- 更适合承载复杂流程与角色分工，便于正式业务协作落地。
- 后台治理、状态看板、协作留痕在大屏端表达更清晰。
- 便于后续承接 AI 匹配入口与规则配置能力。

## 2. 角色目标与权限划分

### 2.1 竹农（供应侧）
**核心目标**
- 高效发布可交易竹材资源。
- 快速响应询价并达成交易。
- 按约履约并建立可信记录，提高后续成交概率。

**关键权限**
- 资源发布/编辑/上下架（含批次、产地、规格、可供时间、含水率/等级等字段）。
- 查看审核状态与驳回原因，补充材料后重提。
- 接收、回复、协商询价（含版本记录）。
- 订单履约更新（备货、发货、到货确认申请等）。
- 上传履约凭证（运输单、质检单、现场照片）。
- 发起/响应争议与补充证据。
- 查看自身信用档案（履约率、响应时效、争议率）。

### 2.2 竹企（需求侧）
**核心目标**
- 快速筛选满足生产需求的资源。
- 与可信供应方高效确认价格与交付条件。
- 低摩擦跟踪履约，降低异常与纠纷成本。

**关键权限**
- 浏览与筛选已审核资源（按地区、品类、规格、供货周期、信誉等级）。
- 发起询价、管理询价会话、对比多家报价。
- 订单确认（数量、单价、交付节点、验收规则）。
- 履约跟踪与节点确认（发货、在途、签收、验收）。
- 异常上报（延迟、质量偏差、数量不符）。
- 争议协同与结果确认。
- 对交易对象进行信用评价。

### 2.3 管理员（平台治理侧）
**核心目标**
- 保障资源真实性、流程规范性与平台规则可执行。
- 在纠纷中提供可追溯、公平、可复核的处理机制。
- 通过治理指标持续降低平台协作摩擦。

**关键权限**
- 资源审核（通过/驳回/补充材料）。
- 用户与资质管理（认证状态、风险标记、权限调整）。
- 争议处置（证据审阅、责任判定、仲裁结论留档）。
- 规则维护（评价规则、违规处理、预警阈值）。
- 治理看板（异常率、争议率、审核时效、履约健康度）。
- AI 匹配后台入口（仅配置与监控入口，算法细节后置）。

## 3. Web 版 MVP 范围

### 3.1 必做范围（MVP）
1. **资源发布与审核**：结构化发布、状态追踪、审核闭环。  
2. **供需匹配与询价**：筛选、询价会话、报价版本留存。  
3. **订单状态流转**：从“待确认”到“已完成/争议中”的标准状态机。  
4. **履约协作**：关键节点更新 + 凭证上传 + 异常预警。  
5. **争议处理**：发起、举证、处理、结论归档。  
6. **平台治理基础**：用户管理、规则说明、信用评价展示。

### 3.2 暂不纳入（但预留）
- 在线支付与结算系统（先做“线下结算 + 凭证登记”）。
- 深度物流系统打通（先做手动节点更新）。
- AI 匹配算法实现（先做推荐位与解释位）。

## 4. 整体信息架构（IA）

### 4.1 顶层站点结构（登录后按角色展示）
- 首页/工作台
- 资源中心
- 询价协同
- 订单与履约
- 争议与仲裁
- 信用与评价
- 规则中心
- 管理后台（管理员可见）

### 4.2 导航策略
- **主导航（左侧）**：承载流程主干（资源→询价→订单→争议→信用）。
- **二级导航（页内 Tab）**：区分状态（如“待审核/已通过/驳回”）。
- **全局状态提醒（顶部）**：待处理事项、异常预警、系统公告。

### 4.3 权限可见性
- 竹农与竹企看到相同流程骨架，但模块入口按角色差异化命名。
- 管理员拥有独立治理模块及跨流程检索能力。

## 6. 各页面核心模块与主要功能

### 6.1 公共首页
- 平台定位区：突出“快速匹配与互信达成”。
- 机制说明区：用流程图展示 6 大机制（审核、询价、履约、争议、信用、治理）。
- 角色入口区：竹农/竹企/管理员快速进入。
- 可信背书区：数据指标（平均响应时长、履约完成率、争议解决时效）。

### 6.2 角色工作台
- 今日待办卡片：待审核反馈、待回复询价、待确认订单、待处理异常。
- 流程进度看板：按状态分栏（进行中/风险中/已完成）。
- 快捷操作：发布资源、发起询价、提交凭证、发起争议。
- 风险提示：超时未响应、凭证缺失、节点延误。

### 6.3 资源列表页
- 高级筛选：产地、品类、竹龄/规格、库存量、可交付时间、认证状态、信用等级。
- 列表卡片：核心字段 + 最近更新时间 + 供应方信誉摘要。
- 批量对比：对比多个资源的关键交易条件。

### 6.4 资源发布/编辑页（竹农）
- 结构化表单：避免自由文本导致信息不完整。
- 资料上传：图片、检测报告、地块/来源说明。
- 草稿与版本：支持草稿保存、历史版本查看。
- 提交审核：提交后进入“待审核”并可追踪状态。

### 6.5 资源详情页
- 资源基础信息模块。
- 交易条件模块（最小起订量、报价区间、交付方式）。
- 可信信息模块（历史履约率、争议记录摘要）。
- 操作区：发起询价/收藏/分享给团队。

### 6.6 资源审核页（管理员）
- 队列视图：按风险级别与提交时间排序。
- 审核面板：资料完整性检查、规则命中提示。
- 处理动作：通过/驳回/要求补件，并写明理由模板。

### 6.7 询价会话页
- 会话时间线：每轮报价与条件变更留痕。
- 版本对比：展示“价格、数量、交付日期、验收标准”的差异。
- 协作备注：团队内部备注与外部可见备注分离。
- 确认动作：达成一致后一键生成订单草案。

### 6.8 订单详情页
- 状态机可视化：待确认→待备货→运输中→待验收→已完成/争议中。
- 条款快照：锁定最终报价版本，防止口径不一致。
- 双方确认记录：确认时间、操作者、变更原因。

### 6.9 履约协作页
- 节点任务列表：每个节点责任方、截止时间、完成标准。
- 凭证中心：运输单、收货单、质检结果统一归档。
- 异常预警：节点逾期、质量偏差、数量偏差自动标记。

### 6.10 争议处理页
- 争议工单：争议类型、触发节点、金额影响。
- 证据链：双方证据时间戳与版本留存。
- 平台处理流：受理→核查→判定→归档，结果可复核。

### 6.11 信用档案页
- 主体画像：履约率、平均响应时长、争议率、处理配合度。
- 评分构成：让评分可解释，避免黑盒。
- 历史记录：近期交易评价与异常处置记录。

### 6.12 规则中心页
- 平台规则公开：审核规范、履约规范、争议处理规范。
- FAQ：常见场景判定说明。
- 规则更新日志：版本化透明发布。

### 6.13 管理后台总览页
- 治理指标看板：审核时效、异常率、争议关闭时长。
- 风险名单：高争议主体、频繁超时主体。
- 运维入口：公告发布、阈值配置、规则更新。

### 6.14 AI 匹配入口页（预留）
- 推荐结果卡片（先模拟数据）。
- 推荐原因解释区（供需匹配维度说明）。
- 人工复核入口（管理员可介入）。

## 7. 主流程设计（按角色）

### 7.1 竹农主流程
1. 登录工作台，查看待处理询价和履约提醒。  
2. 新建资源，补充结构化信息并提交审核。  
3. 审核通过后资源进入可见池，接收竹企询价。  
4. 在询价会话中协商价格/数量/交付节点，形成最终版本。  
5. 确认订单后按节点更新履约状态并上传凭证。  
6. 若出现异常，先协商；无法达成一致则进入争议流程。  
7. 交易完成后获得评价，沉淀信用资产。

### 7.2 竹企主流程
1. 在资源中心按需求筛选供应资源。  
2. 进入资源详情，查看可信信息后发起询价。  
3. 多方比价并确认最优方案，生成订单并确认条款。  
4. 跟踪履约节点，对发货、到货、验收进行确认。  
5. 发现异常时提交问题并附证据；必要时发起争议。  
6. 交易完成后进行评价，优化后续供应商选择。

### 7.3 管理员主流程
1. 审核新发布资源，保证信息真实性和完整性。  
2. 监控询价与订单流程中的异常信号（超时、偏差、重复争议）。  
3. 受理争议工单并组织证据核查，输出判定结果。  
4. 维护规则与阈值，定期发布治理公告。  
5. 复盘治理数据，优化审核与争议策略。

## 8. AI 智能匹配预留位置与展示建议（不含实现）

### 8.1 前台预留
- 资源列表顶部：`为你推荐（智能匹配）` 区块。
- 需求发布后：展示“可能匹配的供应资源 Top N”。
- 推荐原因标签：如“规格匹配度高 / 距离近 / 历史履约稳定”。

### 8.2 询价环节预留
- 在询价会话侧边栏增加“相似成交参考”模块（仅作辅助）。
- 提醒用户“推荐仅供参考，以双方确认为准”。

### 8.3 管理后台预留
- AI 匹配引擎入口：查看推荐命中率、人工采纳率、异常反馈。
- 模型治理占位：可配置“解释展示开关”“行业规则优先级”（仅 UI 预留）。

### 8.4 交互原则
- **可解释**：每条推荐要给出可读原因。
- **可干预**：允许人工覆盖推荐结果。
- **可追踪**：记录推荐结果与最终成交差异，用于后续迭代。

## 9. 视觉风格建议（B 端/产业互联网）

### 9.1 总体风格
- 色彩：低饱和绿色为主（主色 + 中性色灰阶 + 大面积白底）。
- 排版：栅格清晰、留白充足、强调信息层级。
- 组件：稳定卡片、线性图标、轻阴影、弱动效。
- 气质关键词：calm / trustworthy / modern / enterprise。

### 9.2 首页设计建议
- 顶部 Hero 简洁表达价值，不用夸张营销文案。
- 使用流程分段图替代炫酷动画，突出机制可信。
- 数据指标采用小卡片矩阵，体现“可治理、可量化”。

### 9.3 列表页设计建议
- 左侧筛选 + 右侧结果（或顶部筛选条 + 卡片列表）。
- 强调“筛选效率”和“关键字段可比性”。
- 支持固定表头/字段排序，便于专业用户连续操作。

### 9.4 详情页设计建议
- 上方固定核心摘要（状态、信誉、关键条款）。
- 下方分区块：资源信息、履约记录、证据材料、操作区。
- 重要决策信息（验收标准、交付节点）视觉加重。

### 9.5 后台页设计建议
- 深浅对比适中，避免高饱和告警色滥用。
- 看板与队列并存：上层看指标，下层看待办。
- 对“异常/争议”使用统一语义色与统一图标，降低认知负担。

## 附：MVP 成功判断（上线验收口径）
- 是否能在 5 分钟内完成“资源发布→询价→订单确认”的标准业务链路。
- 是否能清晰看到每个关键节点的责任方与状态。
- 是否能追溯报价版本、履约凭证、争议证据。
- 是否能通过信用与规则页面体现“互信达成”机制。
